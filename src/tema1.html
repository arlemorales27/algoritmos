Algoritmos y Lógica Computacional

Una guía completa para entender los fundamentos de la programación y el diseño de algoritmos

1. ¿Qué es el software?

El software es un conjunto de instrucciones o programas que le indican a una computadora qué hacer. Es el componente intangible de un sistema informático, a diferencia del hardware, que es el componente físico.

2. Tipos de software

Software Web

Aplicaciones que se ejecutan en un navegador, como Google Chrome o Firefox. Ejemplo: Gmail.

Software de Escritorio

Programas instalados en una computadora, como Microsoft Word o Visual Studio Code.

Software Móvil

Aplicaciones diseñadas para dispositivos móviles, como WhatsApp o juegos en Android/iOS.

3. Lenguajes de programación

Un lenguaje de programación es un medio para comunicarse con las computadoras mediante instrucciones. Los lenguajes más populares dependen del objetivo:

  • Web: JavaScript, HTML, CSS
  • Móvil: Swift (iOS), Kotlin (Android)
  • Escritorio: C#, Java
  • Datos/IA: Python, R

4. ¿Qué es un algoritmo?

Un algoritmo es una secuencia de pasos finitos y ordenados para resolver un problema. Ejemplo: una receta de cocina es un algoritmo.

5. Tipos de algoritmos y ejemplos

  • Búsqueda: Búsqueda binaria para encontrar un elemento en una lista ordenada.
  • Ordenamiento: Algoritmo de burbuja para ordenar una lista.
  • Recursivos: Cálculo del factorial de un número.

6. Representación de un algoritmo

Pseudocódigo

Inicio
    Leer número
    Si número > 0 entonces
        Escribir "Número positivo"
    Sino
        Escribir "Número no positivo"
Fin

Diagrama de flujo

Un diagrama de flujo usa formas (como óvalos, rectángulos y diamantes) para representar pasos, decisiones y flujos.

7. Pasos para resolver un problema

  1. Entender el problema.
  2. Definir entradas y salidas.
  3. Diseñar el algoritmo.
  4. Probar el algoritmo con casos.
  5. Implementar en un lenguaje de programación.
  6. Depurar y optimizar.

8. ¿Qué es JavaScript?

JavaScript es un lenguaje de programación utilizado principalmente para crear interactividad en páginas web. Es versátil y se usa en front-end, back-end (con Node.js) y más.

9. Editores de código e IDEs

Editores de código: Visual Studio Code, Sublime Text.

IDEs: IntelliJ IDEA, Eclipse.

Editores con IA: GitHub Copilot, Cursor, que sugieren código automáticamente.

10. Herramientas NoCode

Plataformas NoCode permiten crear software sin programar: Bubble, Webflow, AppGyver.

11. Ecosistema de un lenguaje de programación

El ecosistema incluye bibliotecas, frameworks, herramientas de compilación, empaquetadores (como npm para JavaScript) y comunidades de soporte.

12. ¿Qué es un framework?

Un framework es un conjunto de herramientas y bibliotecas que facilita el desarrollo. Ejemplos: React (front-end), Django (back-end).

13. Backend

El backend es la parte del software que maneja la lógica, bases de datos y servidores. Ejemplo: Node.js, Python Flask.

14. Frontend

El frontend es la interfaz que el usuario ve e interactúa. Usa HTML, CSS y JavaScript.

15. Protocolo HTTP

HTTP (HyperText Transfer Protocol) es el protocolo para transferir datos en la web. Incluye peticiones (GET, POST) y respuestas (200 OK, 404 Not Found).

16. Servidor web

Un servidor web es un software o hardware que responde a peticiones HTTP, sirviendo contenido como páginas web. Ejemplo: Apache, Nginx.

17. Hosting y Dominio

Hosting: Servicio que almacena tu sitio web en servidores. Ejemplo: AWS, Hostinger.

Dominio: La dirección de tu sitio web, como www.ejemplo.com.

18. Ejercicio Práctico

Instrucciones Generales

  • Para cada problemática, seguir el paso a paso de resolución de problemas.
  • Dividir el problema en partes o módulos si es necesario.
  • Plantear dos alternativas de solución.
  • De la solución escogida, elaborar un mockup de la posible aplicación (puede usar herramientas como Figma, Sketch, o Google Stitch AI).
  • Hacer un algoritmo sobre una temática específica tanto en pseudocódigo como en diagrama de flujo.
  • Consultar qué es un pitch y vender el producto a todos los integrantes del grupo.

Situación 1: Gestión de Recolección de Basuras en Armenia, Quindío

El sistema actual de recolección de basuras en Armenia, Quindío, presenta deficiencias que afectan a la población. Los camiones recolectores operan con rutas y horarios desactualizados, no dando abasto con el crecimiento urbano y el aumento de residuos. Los ciudadanos reportan acumulación de basuras, desbordamiento de contenedores, y retrasos, especialmente en zonas comerciales y residenciales de alta densidad. La situación empeora en fines de semana y temporada turística. Los barrios periféricos sufren servicios irregulares, generando vertederos improvisados. La falta de monitoreo en tiempo real y la descoordinación entre rutas causan ineficiencias.

Situación 2: Sistema de Transporte Público en Armenia, Quindío

El transporte público en Armenia carece de información centralizada sobre rutas y horarios. Los usuarios enfrentan dificultades para planificar viajes debido a la ausencia de mapas actualizados. Los paraderos no tienen información visible sobre frecuencias ni destinos, y la señalización es inadecuada. Turistas y nuevos residentes dependen de consultas informales. Las empresas transportadoras no tienen un sistema unificado, y los cambios en rutas o frecuencias no se comunican efectivamente, afectando la movilidad urbana.

Situación 3: Gestión de Servicios en Consultorio Veterinario "Huellitas Saludables"

El consultorio veterinario Huellitas Saludables presenta deficiencias en la gestión de servicios. Los tiempos de espera son prolongados debido a un sistema ineficiente de agendamiento. El registro manual de historias clínicas genera retrasos y dificulta el seguimiento de tratamientos. Los clientes tienen problemas para agendar citas de control y urgencias. La comunicación entre el personal médico y los dueños es limitada, causando confusiones. El inventario de medicamentos se maneja de forma desorganizada, provocando desabastecimiento.

Situación 4: Gestión de Aforo en el Estadio Centenario y Coliseo del Café

El Estadio Centenario y el Coliseo del Café enfrentan problemas en el control de aforo durante eventos. Los accesos congestionados generan largas filas, y la falta de un sistema automatizado de conteo en tiempo real impide conocer la ocupación exacta. Se reportan casos de sobreventa de boletos e ingresos no autorizados. La distribución desorganizada de espectadores causa aglomeraciones, y el personal de seguridad carece de herramientas para monitorear la capacidad por sectores y coordinar emergencias.

Situación 5: Gestión Sostenible de Recursos en la Universidad Alexander Von Humboldt

La Universidad Alexander Von Humboldt enfrenta un desafío en la gestión sostenible de recursos educativos y uniformes. Los estudiantes descartan libros académicos, uniformes y camisetas en buen estado al finalizar semestres o graduarse. La falta de una plataforma para reutilizar estos materiales genera desperdicio y costos innecesarios. Los libros de texto representan una inversión significativa, pero terminan almacenados o descartados, contradiciendo los principios de sostenibilidad de la universidad.
