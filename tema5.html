¿Por qué funciones?

Una función encapsula instrucciones bajo un nombre para poder reutilizarlas, organizar el código y evitar repeticiones. En esta guía no usamos arreglos para que el foco esté en los fundamentos.

1) ¿Qué es una función en JavaScript?

Definición: Bloque de código nombrado que puedes invocar para ejecutar una tarea. Puede recibir parámetros (datos de entrada) y retornar un valor.

// Declarar y usar una función
function saludar(){
  console.log("Hola, Dev!");
}
// invocación
saludar();
  • Declaración: define el bloque de código.
  • Invocación: ejecuta la función por su nombre.
  • Alcance: variables internas no se ven fuera.

2) Sintaxis de una función (declaración)

Sintaxis 
function nombre(param1, param2) {
  // instrucciones
  return valor;
}
// Área de un rectángulo (param y retorno)
function areaRect(base, altura){
  return base * altura;
}
console.log("Área:", areaRect(8, 5));

// Buenas prácticas
// 1) Un nombre claro describe la acción
// 2) Una función = una responsabilidad
// 3) Retorna datos, no prints (cuando debas reutilizar el resultado)

3) Funciones sin parámetros y sin retorno

Ejecutan una acción y no devuelven valor.

// Notifica saldo bajo
function alertaSaldoBajo(){
  console.log("⚠️ Saldo por debajo del mínimo");
}
alertaSaldoBajo();

// Imprime cabecera de reporte
function cabecera(){
  console.log("==== Reporte Diario ====");
}
cabecera();

4) Funciones con parámetros y con retorno

Reciben datos de entrada y devuelven un resultado.

// Calcula IVA y total
function totalConIVA(subtotal, iva){
  const impuesto = subtotal * iva;
  const total = subtotal + impuesto;
  return total; // retorna el total
}
console.log("Total:", totalConIVA(85000, 0.19));

// Convierte de Celsius a Fahrenheit
function cToF(c){
  return (c * 9/5) + 32;
}
console.log("32°C son", cToF(32), "°F");

6) Ejemplo práctico: Cajero Automático

Simulación mínima de un cajero con funciones pequeñas. Sin arreglos, todo con variables simples.

// Cajero: consultar, depositar, retirar
let saldo = 150000; // COP

function consultar(){
  console.log("Saldo:", saldo, "COP");
}

function depositar(monto){
  if (monto <= 0) return "Monto inválido";
  saldo = saldo + monto;
  return "Depósito OK. Nuevo saldo: " + saldo + " COP";
}

function retirar(monto){
  if (monto <= 0) return "Monto inválido";
  if (monto > saldo) return "Fondos insuficientes";
  saldo = saldo - monto;
  return "Retiro OK. Nuevo saldo: " + saldo + " COP";
}

consultar();
console.log(depositar(25000));
console.log(retirar(70000));
consultar();
  • Estado: la variable saldo vive fuera y se modifica.
  • Validaciones: evita montos inválidos y sobregiros.
  • Reutilización: cada acción es una función independiente.

7) Función flecha: ¿qué es? + ejemplos

Una función flecha es una forma abreviada de crear funciones. No cambia sola el this y permite retorno implícito cuando el cuerpo es una sola expresión.

Patrones comunes (sin arrays) 
// 1) Sin parámetros, sin retorno
const sonar = () => { console.log("🔔 Ding!"); };
sonar();

// 2) Con parámetros y retorno implícito
const cuadrado = x => x * x; // una sola expresión
console.log("9^2 =", cuadrado(9));

// 3) Con bloque y return explícito
const convertir = (monto, tasa) => {
  const total = monto * tasa;
  return total;
};
console.log("Total:", convertir(100000, 1.19));

// 4) Retornar un objeto literal (sin arrays)
const crearTicket = (codigo, total) => ({ codigo: codigo, total: total });
const t = crearTicket("A-01", 55000);
console.log("Ticket:", t.codigo + ", " + t.total + " COP");
